## **4.** Comunicação HTTP
>Em HTTP o que é requisição como é o termo em inglês?
```
É uma solicitação do cliente.
 Em inglês o termo se chama request
```

>Em HTTP o que é resposta como é o termo em inglês?
```
É a resposta que o sistema da em relação a request.
 O termo em inglês se chama response
```

>Em HTTP o que é o corpo da mensagem como é o termo em inglês?
```
É a parte que carrega os dados da request ou da response. 
O termo se chama message body
```

## **5.** JSON

>O que é JSON e qual é sua estrutura?
```
JSON (JavaScript Object Notation) é um formato utilizado para armazenar e trocar informações entre sistemas, sendo muito comum em APIs e aplicações web. Ele organiza os dados em pares de chave e valor, o que facilita a leitura tanto por pessoas quanto por computadores.

Sua estrutura é composta por objetos, delimitados por chaves {}, e listas (arrays), delimitadas por colchetes []. Dentro dos objetos, cada informação é representada por uma chave e um valor, separados por dois pontos (:), e os diferentes pares são separados por vírgulas.
```

## **6.** Python backend

>Como se chama o arquivo que temos que criar assim que começamos um projeto python? e para que serve? e como executá-lo?
```
O arquivo que criamos ao iniciar um projeto em Python chama-se main.py.

Ele serve como o arquivo principal do projeto, sendo o ponto de entrada da aplicação. É nele que o programa começa a ser executado e onde geralmente fica a função principal ou as chamadas para os demais módulos do projeto

Para executá-lo, basta abrir o terminal na pasta do projeto e utilizar o comando:

python main.py
```
